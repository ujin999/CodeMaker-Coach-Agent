# Counter-example / Corner-case Search (반례 및 모퉁이 케이스 탐색 패턴)

- **Korean Keywords**: 반례 탐색, 에지 케이스, 오답 원인 분석, 예외 조건, 정답 코드 미노출
- **English Keywords**: Counter Example, Edge Case, Failure Analysis, Exception Boundary, Hidden Testcase

## Concept Summary
반례는 코딩 테스트 풀이자가 작성한 그릇된 알고리즘 논리를 저격하여 거짓임을 입증하는 구체적인 입력 세트입니다. 주로 비정상적이거나 극단적인 입력 경계, 빈 데이터, 중복된 값의 유입 등으로 구성됩니다.

## When to Use
- 사용자가 작성한 코드가 예제 테스트케이스는 완벽히 통과하나 숨겨진 테스트케이스에서 실패(WA)할 때 제공합니다.
- 사용자를 직접 돕기 위해 정답 코드를 보여주지 않고 스스로 버그를 고칠 수 있는 최소 규모의 반례를 탐색해 줄 때 활용합니다.

## Representative Corner Cases for Algorithms
- **정수 범위**: 입력값이 0인 경우, 최댓값 조건, 오버플로우가 나기 직전의 극대값, 음수 포함 유무.
- **배열/문자열**: 비어 있는 경우(Length = 0), 길이가 1인 경우, 모든 원소가 동일한 값인 경우.
- **그래프**: 노드가 1개인 경우, 서로 단절된 컴포넌트가 존재하는 경우, 연결이 일직선 트리 형태(Degenerate Graph)로 구성된 경우.
- **그리디**: 직관적 규칙을 무너뜨리는 미세한 순서 뒤집힘 조합 케이스.

## Common Mistakes
- 일반적인 케이스(Normal cases)만 고려하고 최악의 크기, 예외 수치(0, 공백)를 대비하지 않고 배열 인덱스나 연산 처리를 시도하는 경우.
- 반례를 알려줄 때 정답 코드의 일부분을 주석이나 텍스트로 흘려버려, 힌트 가이드의 보안 정책을 무너뜨리는 경우.

## Problem Generation Guidance
- 문제 서술란에 예외적 상황(예: "단, 만족하는 경우가 없을 경우 -1을 출력한다")을 명문화하여, 조건문 처리가 들어갈 자리를 마련해 주십시오.

## Testcase Generation Guidance
- 일반 예제 이외에도, 무조건적인 실패를 내는 극단적인 에지 조건들을 히든 테스트케이스 전체 풀에 50% 이상 반영해야 완벽한 채점이 됩니다.

## Hint Guidance
- 1단계: 빈 입력, 혹은 입력의 크기가 1인 극단적인 최솟값 상황에서도 작성한 코드의 조건이 정상으로 흐르는지 따져보게 합니다.
- 2단계: 값이 0이거나 음수인 경우, 그리고 동일한 중복 데이터가 연속해서 들어오는 등의 특수한 배치를 상상해보라고 유도합니다.
- 3단계: 정답 코드는 노출하지 않은 상태에서, 오류를 일으키는 가장 작은 단위의 입력 배열 형태를 반례 템플릿(e.g., N=1, Data=[0])으로 설명합니다.
