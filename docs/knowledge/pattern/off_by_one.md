# Off-by-One Errors (경계값 인덱스 오류 패턴)

- **Korean Keywords**: 인덱스 범위 초과, 경계 조건, 등호 실수, 루프 범위 설정 오류
- **English Keywords**: Off-by-one, Index Out of Bounds, Boundary Condition, Inequality Error, Loop Range

## Concept Summary
Off-by-One 에러는 반복문(Loop)의 횟수 지정, 슬라이싱 조건 범위 설정, 배열 인덱스 연산 시 시작/끝 경계값의 범위 판단 착오로 인해 1만큼 더 많거나 적게 반복·접근하여 발생하는 흔하고 고질적인 프로그래밍 논리 버그입니다.

## When to Use
- 예제 입력은 잘 통과하는데 실제 숨겨진 채점 케이스에서 `IndexError` 또는 미세한 값 차이로 오답(`WA`)을 받을 때.
- 이분 탐색 범위(low ~ high) 결정이나 투 포인터 경계 조건 제어를 설명하고 유도할 때.

## Common Mistakes
- 0-based 인덱스를 쓰는 언어에서 크기가 $N$인 배열의 마지막 요소 인덱스를 $N$으로 접근하는 경우.
- `range(0, N)`이 $0$부터 $N-1$까지만 도는 사실을 망각하여 루프 마지막 값을 유실하는 경우.
- 문자열 또는 구간 탐색 시 시작 인덱스와 종료 인덱스의 등호 포함 여부($\le$ vs $<$)를 불일치하게 설계하는 경우.

## Problem Generation Guidance
- 문제 제약의 경계에 있는 값들을 의도적으로 입력에 섞어 넣게끔 요구사항을 고정하십시오. (예: 1부터가 아닌 0부터 시작하는 카운팅 등)
- 경계 조건을 디테일하게 체크해야 하는 세부 조건문(예: 배열의 양 끝에서의 인덱스 전진)을 요하도록 제약 조건을 명시합니다.

## Testcase Generation Guidance
- 길이가 1인 입력, 인덱스가 0일 때의 조건, 마지막 인덱스일 때의 조건을 모두 에지 케이스로 작성해야 합니다.
- 조건의 경계에 아슬아슬하게 맞닿아 통과 또는 거절되는 임계값(Threshold) 입력 조합을 준비해 주십시오.

## Hint Guidance
- 1단계: 인덱스의 양 끝값(0번째, N번째) 처리에 실수가 없는지 꼼꼼하게 들여다보도록 촉구합니다.
- 2단계: 등호 포함 여부($<=$ 와 $<$)를 하나씩 디버깅해 보며, 배열 크기가 아주 작을 때($N=1$)의 거동을 직접 그려보라고 합니다.
- 3단계: 루프의 시작점과 끝부분 인덱스가 접근 가능한 최대 영역을 넘지 않도록 제한하는 보조 가드 조건 설정을 뼈대로 제시합니다.
