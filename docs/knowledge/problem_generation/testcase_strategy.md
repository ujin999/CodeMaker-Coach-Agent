# Testcase Generation Strategy (테스트케이스 생성 전략)

- **Korean Keywords**: 테스트케이스 생성, 샘플 케이스, 히든 케이스, 에지 케이스, 반례 검증
- **English Keywords**: Testcase Strategy, Sample Testcase, Hidden Testcase, Edge Testcase, Validator

## Concept Summary
채점 시 사용할 테스트케이스를 다양하게 마련하여 올바르지 않은 풀이(시간 초과 코드, 예외 누락 코드 등)를 완전하게 필터링할 수 있도록 돕는 테스트케이스 배치 설계 규칙입니다.

## Testcase Categorization
1. **예제 (Sample) 테스트케이스**:
   - 사용자가 문제를 읽고 바로 손으로 디버깅해볼 수 있는 직관적이고 작은 크기의 입력 세트(2~3개).
   - 문제 지문에 반드시 상세한 출력 유도 설명이 적혀 있어야 합니다.
2. **숨겨진 (Hidden) 테스트케이스**:
   - 일반적인 유효 입력 범위의 데이터들로 골고루 분포된 세트(3~5개).
   - 정답 코드의 정상 동작 여부 및 미세한 분기 논리의 완결성을 검증합니다.
3. **에지 (Edge) 테스트케이스**:
   - 데이터 크기의 최대 한계치($N=N_{max}$), 데이터 크기의 최솟값($N=1$ 또는 $0$), 모든 입력 원소가 동일한 값, 극대 음수/양수 정수 범위 등의 한계 조건을 테스트합니다.
   - 시간 초과(TLE)와 오버플로우를 잡아내기 위한 강력한 도구입니다.

## Testcase Generation Guidance
- 수작업으로 지어낸 단순한 값 대신, 조건의 경계값과 최대 크기 세트를 꼭 1개 이상 배치하여 $O(N^2)$ 코드를 저격하게 만드십시오.
- 입력값 포맷(공백 구분, 개행 구분 등)이 문제의 "입력 형식" 서술과 한 자의 오차도 없이 완벽히 일치해야 합니다.
