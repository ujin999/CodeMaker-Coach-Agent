# Validator Checklist (생성 문제 검증 체크리스트)

- **Korean Keywords**: 검증 체크리스트, 예제 일치 여부, 제약조건 검증, 솔루션 검증, 재생성 트리거
- **English Keywords**: Validator Checklist, Self-Consistency, Executable Verification, Regeneration Trigger

## Concept Summary
생성된 문제가 학습자에게 제공되기 전, 기계적으로 오류가 없으며 문제 지문과 테스트케이스 간에 모순이 발생하지 않았는지 다방면으로 확인하는 검증 프로토콜입니다.

## Checklist Items
1. **입출력 데이터 일치성**:
   - 지문에 제공된 예제 입력값들을 Reference Solution(정답 코드)에 넣고 실행했을 때의 결과 출력이, 지문에 적힌 예제 출력과 공백/줄바꿈 문자를 포함하여 정확히 일치하는지 판정합니다.
2. **제약조건 부합 여부**:
   - 생성된 테스트케이스들의 원소들이 문제 지문에서 천명한 숫자 범위($1 \le X \le 10,000$ 등)와 크기를 지키고 있는지 검사합니다.
3. **난이도 및 복잡도 부합성**:
   - 요구하는 시간복잡도(예: $O(N \log N)$)로 구현된 코드가 데이터 최댓값 에지 케이스에서 시간제한(예: 1초) 내에 들어오는지 논리적으로 또는 샌드박스로 검증합니다.
4. **문제 정보의 명확성**:
   - 입력과 출력의 포맷 및 자료형 제한(정수, 실수, 공백 분리 등)이 불분명하거나 빼먹은 부분이 없는지 판별합니다.

## Action on Failure
- 위 항목 중 단 하나라도 실패(Fail)할 시, Validator는 실패 사유를 구체적으로 피드백하고 **문제 재생성(Regeneration)** 루프를 태웁니다.
