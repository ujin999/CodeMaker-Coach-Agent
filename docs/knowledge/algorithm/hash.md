# Hash (해시)

- **Korean Keywords**: 해시, 딕셔너리, 해시 맵, 중복 제거, 키-값 쌍, 탐색 효율성
- **English Keywords**: Hash Map, Dictionary, Set, Hash Function, Collision, Key-Value, O(1) Lookup

## Concept Summary
해시 자료구조는 임의 크기의 데이터를 고정된 크기의 값(해시 코드)으로 매핑하여 데이터를 관리하고, 평균 $O(1)$의 압도적인 속도로 데이터의 삽입, 검색, 삭제를 수행할 수 있게 하는 기법입니다.

## When to Use
- 대량의 데이터셋에서 특정 데이터의 존재 여부를 초고속으로 조회해야 할 때 (Set 활용).
- 문자열이나 식별자를 인덱스로 삼아 정보(빈도 수, 짝궁 정보 등)를 관리해야 할 때 (Map 활용).
- 정렬 조건이 없어도 데이터의 짝을 신속히 찾는 단방향 조회가 필요할 때.

## Common Mistakes
- 언어별로 해시 충돌(Hash Collision)이 심하게 일어날 수 있는 에지 입력에서 $O(1)$이 $O(N)$으로 악화될 수 있음을 무시하는 경우.
- 키 탐색 시 키가 존재하지 않는 경우의 딕셔너리 예외(KeyError 등)를 다루지 않는 경우.
- 동명이인이나 중복 키 처리에 대한 세밀한 로직 부재.

## Time Complexity Guidance
- 데이터 삽입, 검색, 삭제 연산: 평균적으로 $O(1)$ 소요.
- 해시 키 순회나 전체 항목 조사는 $O(N)$ 소요.

## Problem Generation Guidance
- 참가자 목록과 완주자 목록 비교, 회원 정보 매칭, 접두사 중복 여부 확인 등 실용적 테마를 구성하십시오.
- 입력으로 문자열이나 튜플이 많이 들어오며 이를 관계성으로 묶어 확인하는 조건이 좋습니다.

## Testcase Generation Guidance
- 동일한 해시 코드로 매핑될 수 있는 중복 키 대입 케이스, 전혀 키가 겹치지 않는 케이스를 섞어서 제공하십시오.
- 최대 십만 건 이상의 키 탐색 연산을 수행하도록 대용량 입력 범위를 갖는 테스트케이스를 설계해 속도 저하를 식별하십시오.

## Hint Guidance
- 1단계: 값의 조회/검색 빈도가 매우 높다면 배열 대신 탐색 속도가 평균 O(1)인 해시 맵(Hash Map)이나 해시 셋(Hash Set)을 활용하십시오.
- 2단계: 키(Key) 값에 고유 식별자나 문자열을 두고 값(Value)에 발생 빈도나 매칭 데이터를 넣어 묶어 보라고 조언합니다.
- 3단계: 해시 맵 생성 및 조건 체크를 통해 결과 데이터를 취합하는 기본 반복 구문 스켈레톤을 설명합니다.
