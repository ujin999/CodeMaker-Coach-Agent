# Breadth-First Search (너비 우선 탐색)

- **Korean Keywords**: 너비 우선 탐색, 최단 경로, 큐, BFS, 그래프 탐색
- **English Keywords**: BFS, Breadth-First Search, Queue, Shortest Path, Level Order Traversal

## Concept Summary
BFS는 그래프 또는 격자 공간에서 시작 노드로부터 가장 가까운 노드들을 먼저 방문하며 탐색 범위를 넓혀가는 알고리즘으로, 가중치가 없는 그래프에서 최단 경로를 찾는 데 주로 사용됩니다.

## When to Use
- 가중치가 1(혹은 동일)인 그래프에서 최단 경로/거리 또는 최소 연산 횟수를 구해야 할 때.
- 격자 지도 등에서 인접한 영역(4방향/8방향)을 한 단계씩 확장하며 조건에 부합하는 범위를 구해야 할 때.

## Common Mistakes
- 노드를 큐에 삽입(push)할 때 바로 방문 처리(visited = True)를 하지 않아 동일 노드가 중복 방문되어 메모리 초과나 시간 초과를 겪는 경우.
- 격자 탐색 시 경계 범위 조건검사(범위 초과 등)를 생략하는 경우.
- 최단 거리를 누적해야 하는데, 방문 배열에 거리 정보를 기록하는 구조가 잘못된 경우.

## Time Complexity Guidance
- 인접 리스트: $O(V + E)$ (V: 정점 수, E: 간선 수)
- 격자 탐색 ($R \times C$): $O(R \times C)$

## Problem Generation Guidance
- 미로 탈출, 최단 이동 횟수, 바이러스 확산 시뮬레이션 등 시각적이고 직관적인 시나리오를 설계하십시오.
- 다중 시작점에서 동시에 번지는 최적 경로 탐색 등 변형 문제를 설계할 수 있습니다.

## Testcase Generation Guidance
- 빈 격자, 벽으로 막혀 도달 불가능한 격자(출력 -1 또는 불가능 판정), 최대 행렬 크기 케이스를 포함하십시오.
- 시작점과 끝점이 겹쳐진 경우나 이동 경로가 유일한 경우를 테스트하십시오.

## Hint Guidance
- 1단계: 가까운 위치부터 차례대로 탐색하는 BFS와 큐(Queue) 자료구조를 사용할 것을 추천합니다.
- 2단계: 최단 경로의 단계적 탐색을 위해 탐색 시점마다 인접 정점의 방문 유무를 즉시 체크하고 최단 거리를 기록하라고 조언합니다.
- 3단계: 격자 내 4방향 탐색 및 큐에서 pop하여 처리하는 기본 뼈대 코드 흐름을 설명합니다.
