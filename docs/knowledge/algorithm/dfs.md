# Depth-First Search (깊이 우선 탐색)

- **Korean Keywords**: 깊이 우선 탐색, 백트래킹, 재귀, DFS, 완전 탐색
- **English Keywords**: DFS, Depth-First Search, Recursion, Backtracking, Stack

## Concept Summary
DFS는 그래프 탐색 알고리즘으로, 현재 분기에서 도달 가능한 가장 깊은 지점까지 우선적으로 탐색한 후, 막다른 길에 다다르면 이전 상태로 돌아가 다른 경로를 깊게 파고드는 방식입니다. 주로 모든 노드를 방문하거나 백트래킹을 활용한 순열/조합 생성에 활용됩니다.

## When to Use
- 모든 경로를 하나씩 확인하여 특정 조건을 만족하는 답을 결정하거나 탐색해야 할 때.
- 사이클 존재 여부를 판별하거나 위상 정렬 등을 수행해야 할 때.
- 백트래킹(Backtracking)을 활용한 상태 트리 전수 탐색이 필요할 때.

## Common Mistakes
- 재귀 호출 종료 조건(Base Case)이 없거나 부적절하여 RecursionError(재귀 한계 초과)가 발생하는 경우.
- 전역 방문(visited) 상태를 백트래킹 복귀 시점에 올바르게 원복(unmark/visited = False)하지 않는 경우.
- 재귀 깊이가 지나치게 길어짐에도 불구하고 스택 깊이 설정 조절(`sys.setrecursionlimit`)을 빼먹는 경우.

## Time Complexity Guidance
- 인접 리스트: $O(V + E)$
- 상태 트리의 가지가 $K$개이고 최대 깊이가 $N$인 백트래킹: $O(K^N)$

## Problem Generation Guidance
- 퍼즐 맞추기, 부분 집합 탐색, 조합 조합 등의 구조를 갖는 문제를 출제하십시오.
- 단순 그래프 탐색보다는 특정 상태 제약 조건을 부여한 탐색 문제가 적절합니다.

## Testcase Generation Guidance
- 탐색 분기가 매우 많고 깊은 최대 크기 N의 깊이 깊은 트리/일자 그래프를 입력값으로 제공하여 스택 오버플로우가 나지 않는지 확인하십시오.
- 불가능한 조건인 빈 데이터나 도달점이 아예 없는 에지 케이스를 포함하십시오.

## Hint Guidance
- 1단계: 모든 가능성을 끝까지 깊이 있게 파고들며 탐색하는 DFS 또는 백트래킹 방식을 사용해 보라고 제안합니다.
- 2단계: 재귀 호출을 사용할 때 '종료 조건'과 '현재 방문 상태', 그리고 탐색 복귀 시 '방문 취소 처리'를 설계하라고 조언합니다.
- 3단계: 재귀 함수의 인자 구성과 방문 체크/해제 과정을 담은 기본 재귀 뼈대를 설명합니다.
