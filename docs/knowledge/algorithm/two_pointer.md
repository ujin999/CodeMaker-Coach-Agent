# Two Pointer (투 포인터)

- **Korean Keywords**: 투 포인터, 슬라이딩 윈도우, 구간 합, 부분 배열, 선형 탐색
- **English Keywords**: Two Pointer, Sliding Window, Interval Sum, Subarray, Linear Scan, Left Right Pointer

## Concept Summary
투 포인터는 1차원 배열에서 서로 다른 위치를 가리키는 두 개의 포인터(인덱스 변수)를 조건에 맞춰 번갈아 가며 이동시켜 가며 원하는 최적의 구간이나 조합을 찾는 효율적인 탐색 기법입니다.

## When to Use
- 정렬된 배열 안에서 두 수의 합이나 차가 특정 값과 일치하는 구간을 찾아야 할 때.
- 연속된 부분 수열 중 특정 조건(예: 합이 $S$ 이상)을 만족하는 구간을 찾아 해당 길이를 조절해갈 때.
- 중복 연산을 제거하여 $O(N^2)$ 접근을 $O(N)$으로 변환할 수 있는 구조일 때.

## Common Mistakes
- 두 포인터(left, right)의 이동 방향과 경계 도달(index out of bound) 처리를 잘못하여 인덱스 범위를 벗어나는 경우.
- 배열이 정렬되지 않은 채 포인터를 서로 교차시키려 해 규칙이 망가지는 경우.
- 구간 합 조건 검사의 세부 판정 오류로 인해 특정 요소를 건너뛰거나 중복 적용하는 경우.

## Time Complexity Guidance
- 일반적으로 두 포인터가 각각 배열 끝까지 최대 1회씩만 이동하므로 $O(N)$ 해결.
- 정렬이 필요한 문제라면 $O(N \log N)$에 수행됨.

## Problem Generation Guidance
- 수열 속에서 합이 $X$인 쌍 찾기, 가장 긴 연속 부분 문자열 검색, 특정 구간 합 구하기 등의 형태로 설계해 주십시오.
- 입력 리스트의 크기가 매우 커서 이중 반복문으로는 시간 초과가 날 수 있도록 제약을 강화해야 합니다.

## Testcase Generation Guidance
- 타겟 구간이 존재하지 않는 경우, 모든 원소를 더해야만 타겟을 만족하는 경우 등을 에지 케이스로 준비하십시오.
- 배열 크기가 최소인 1 또는 2 크기의 데이터 처리가 문제없이 되는지 검증하십시오.

## Hint Guidance
- 1단계: $O(N^2)$의 연산을 포인터 변수 2개(예: left, right)를 이용해 선형 탐색($O(N)$)으로 바꿀 수 있는지 생각해 봅니다.
- 2단계: 배열을 정렬한 후 시작점과 끝점에 각각 포인터를 두거나, 나란히 같은 방향으로 출발시켜 보라고 조언합니다.
- 3단계: left와 right를 비교하면서 범위를 이탈하지 않고 포인터를 전진시키는 제어 루프 스켈레톤을 구성합니다.
